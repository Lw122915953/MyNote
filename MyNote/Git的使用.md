# 一、代码规范管理的优点

1、开源的分布式的版本控制系统

2、无需服务器支持

3、远程仓库 ：GitHub     码云

# 二、git和SVN

1、git是分布式，SVN不支持分布式

2、git是元数据，SVN是文件

# 三、概念

1、工作区：版本控制的目录

2、暂存区：.git/index   目录    存储新增

3、版本库：.git

# 常用命令

1、git   init   初始化    将当前目录作为工作目录

2、git   add     文件名    将指定文件添加到git的暂存区

3、git   commit  -m   将暂存区的文件添加到版本库

4、git   status   查看暂存区的状态

5、git  diff   查看对比

6、reset   HEAD   撤销暂存区的内容

7、‘rm  删除

8、mv     重命名
